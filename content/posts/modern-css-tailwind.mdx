## Introduction

Tailwind CSS has revolutionized how we write CSS by providing a utility-first approach that speeds up development while maintaining flexibility. In this article, I'll share advanced techniques I use daily.

## Custom Configuration

### Extending the Default Theme

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          50: '#f0f9ff',
          100: '#e0f2fe',
          500: '#0ea5e9',
          900: '#0c4a6e',
        },
      },
      spacing: {
        '128': '32rem',
        '144': '36rem',
      },
      fontFamily: {
        sans: ['Inter', 'sans-serif'],
      },
    },
  },
};
```

## Component Patterns

### Button Component

```tsx
const buttonVariants = {
  primary: 'bg-blue-600 hover:bg-blue-700 text-white',
  secondary: 'bg-gray-200 hover:bg-gray-300 text-gray-900',
  outline: 'border-2 border-blue-600 text-blue-600 hover:bg-blue-50',
};

const buttonSizes = {
  sm: 'px-3 py-1.5 text-sm',
  md: 'px-4 py-2 text-base',
  lg: 'px-6 py-3 text-lg',
};

<button 
  className={`
    ${buttonVariants.primary}
    ${buttonSizes.md}
    rounded-lg
    transition-colors
    duration-200
  `}
>
  Click Me
</button>
```

## Animation Techniques

### Hover Effects

```tsx
<div className="
  group
  relative
  overflow-hidden
  rounded-lg
  transition-transform
  hover:scale-105
  hover:shadow-xl
">
  <img 
    src="/image.jpg"
    className="
      transition-opacity
      group-hover:opacity-75
    "
  />
  <div className="
    absolute
    inset-0
    bg-gradient-to-t
    from-black/60
    to-transparent
    opacity-0
    group-hover:opacity-100
    transition-opacity
  ">
    <p className="
      absolute
      bottom-4
      left-4
      text-white
      font-semibold
      transform
      translate-y-4
      group-hover:translate-y-0
      transition-transform
    ">
      Hover Text
    </p>
  </div>
</div>
```

## Layout Techniques

### Grid Masonry (CSS Grid)

```tsx
<div className="
  columns-1
  md:columns-2
  lg:columns-3
  gap-4
  space-y-4
">
  {items.map(item => (
    <div 
      key={item.id}
      className="break-inside-avoid"
    >
      {/* Content */}
    </div>
  ))}
</div>
```

## Performance Tips

### Purging Unused CSS

Tailwind automatically purges unused styles in production:

```js
// tailwind.config.js
module.exports = {
  content: [
    './app/**/*.{js,ts,jsx,tsx}',
    './components/**/*.{js,ts,jsx,tsx}',
  ],
};
```

### JIT Mode Benefits

- Generate styles on-demand
- Faster build times
- All variants available
- Smaller CSS files
